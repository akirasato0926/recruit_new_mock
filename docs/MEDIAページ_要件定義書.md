## 1. 背景・目的

### 1.1 背景
官公庁・自治体の採用は「公務員＝堅い・情報が少ない・民間と比較しにくい」という認知課題を抱えており、求人ページ単体では志望度形成（魅力理解・働くイメージの具体化）が難しい。記事・動画・インタビュー等のコンテンツで「働く人・現場・キャリア」を継続的に発信し、潜在層〜顕在層を育成（リードナーチャリング）する場が必要となる。

### 1.2 目的
PUBLIC CONNECT MEDIA は、**官公庁・自治体ではたらく人／はたらきたい人に向けたオウンドメディア**として、以下を目的とする。

1. **集客（認知拡大・SEO流入）**：公務員試験・転職・キャリアに関する検索流入を獲得し、新規ユーザーを呼び込む。
2. **志望度形成（態度変容）**：職員インタビュー・動画・コラムで「働くリアル」を伝え、公務員・自治体への志望度を高める。
3. **求人サイト・会員登録への送客（コンバージョン）**：メディア閲覧者を求人サイト／会員登録／個別自治体（PR）へ誘導する。
4. **掲載自治体の採用広報支援（PR枠）**：特集・記事・動画を通じて掲載自治体のブランディングと母集団形成を支援する。

### 1.3 ゴール／KPI（例）
- 集客：オーガニック流入数、検索順位、新規セッション数
- エンゲージメント：回遊率（セッションあたり閲覧記事数）、平均滞在時間、動画再生数・完了率
- 送客：求人サイト遷移率、会員登録CVR、PR記事/自治体ページへの遷移数
- PR：PR枠の表示・クリック・送客実績（掲載自治体向けレポート）

## 2. スコープ

### 2.1 対象範囲（In Scope）
- 求職者向けメディアの **公開（フロントエンド）画面**一式
  - トップページ（メディアホーム）
  - 一覧ページ（カテゴリー別）
  - 記事詳細（インタビュー／コラム／ガイド記事）
  - 動画ページ
  - SEO向けガイドコンテンツ群（公務員試験・転職テーマ）
- コンテンツの分類体系（カテゴリー・キーワードタグ）
- メディア内外への導線（求人サイト・会員登録・PR）
- サイドバー（ランキング・広告枠）

### 2.2 対象外（Out of Scope。別途定義）
- 記事・動画を作成／公開管理する **CMS・編集管理画面**（運用側機能）
- 会員登録・ログイン・マイページ等の **会員基盤**（求人サイト側に準拠）
- 広告配信システム（アドサーバー連携）の内部仕様
- 動画配信基盤（プレイヤー・エンコード・ホスティング）の内部仕様
- 推薦・パーソナライズエンジン

> 上記対象外は依存先として「外部連携」（第10章）で前提のみ定義する。

## 5. コンテンツ要件

### 5.1 コンテンツ種別（カテゴリー）
メディアの一次分類は **「記事」「動画」** の2軸。記事はさらに以下のコンテンツタイプを持つ。

| 大分類 | コンテンツタイプ | 例 |
|---|---|---|
| 記事 | 職員インタビュー | 「窓口の最前線で見つけたやりがい」（神戸市役所） |
| 記事 | 管理職インタビュー | 「管理職が語る組織づくり」（上北山村役場） |
| 記事 | 座談会 | 「心理的安全性が生み出す連帯感」（大洲市役所） |
| 記事 | 技術職インタビュー | 「土木技師として働くという選択」（京都市役所） |
| 記事 | 新卒インタビュー | 「1年目のリアル」（龍ケ崎市役所） |
| 記事 | 編集部コラム | 面接対策・自己PR・試験準備の時期 など |
| 記事 | ガイド記事（SEO） | 公務員試験の内容/流れ/区分/過去問、年代別転職、給与 など |
| 動画 | 動画コンテンツ | 番組型（ふるさと納税、首長、現場、逆面接 等） |

### 5.2 キーワードタグ体系
横断的な絞り込み用に **キーワードタグ**を付与（複数付与可）。タグは固定ではなく、コンテンツの拡充や運用に合わせて追加・拡張していく。以下はトップのキーワードバーに表示するタグの例（現時点のサンプル）：

`#公務員試験 / #移住・定住 / #子育て支援 / #行政DX / #働き方改革 / #土木・建築 / #保育・教育 / #防災 / #医療・福祉 / #地域おこし / #観光・PR / #環境・SDGs / #キャリア採用 / #新卒採用 / #技術職 / #事務職 / #ワークライフバランス / #心理的安全性 / #面接対策 / #インターンシップ`

- 各コンテンツは「大分類（記事/動画）」＋「コンテンツタイプ」＋「キーワードタグ（複数）」を保持する。
- タグはカード上にチップ表示され、クリックで該当タグの絞り込みに連動する。

### 5.3 特集（FEATURE）
トップ最上部のヒーローで訴求する編集主導の枠。
- 通常特集（例：離島で働く特集、自治体インターンシップ特集、キャリアフェス告知、面接対策、動画訴求）
- **PR特集**：掲載自治体の有料枠。`PR` ラベルを明示（例：相模原市役所 新卒採用／尼崎市役所 DX職／新潟県庁 転職者インタビュー）。

## 6. 機能要件（画面別）

### 6.1 トップページ（メディアホーム / index.html）

#### 6.1.1 ヘッダー（グローバル）
- ロゴ／ブランド名「PUBLIC CONNECT MEDIA」（トップへのリンク）
- グローバルナビ：`特集 / インタビュー / 動画 / コラム`（同一ページ内アンカー遷移）
- 記事検索ボックス（キーワード入力）
- 「求人サイト」への導線、「会員登録」CTA
- 要件：上部固定（スクロール追従）を推奨。スマホではナビを集約（ハンバーガー等）。

#### 6.1.2 ヒーロー特集スライドショー（FEATURE）
- 複数バナー（初期8枚）を**自動スライド（5秒間隔）**で巡回。
- **無限ループ**：前後のスライドが左右に見切れて表示される中央寄せ表示。
- 操作：前へ／次へ矢印、ドットインジケーター（クリックで該当スライドへ）。手動操作後はオートプレイをリスタート。
- 各スライドは画像＋遷移先リンク。PR枠には `PR` ラベルを重畳表示。
- レスポンシブでスライド幅・見切れ量が可変。

#### 6.1.3 絞り込み（フィルタ）
- **カテゴリーフィルタ**：`すべて / 記事 / 動画`。選択カテゴリーに該当しないセクション・カードを非表示。
- **キーワードフィルタ**：キーワードピル（タグは追加・拡張される。表示数・並び順は運用で調整）を**複数選択（OR 条件＝いずれか一致で表示）**。
- カード内に自動付与されたタグチップ（大分類＋キーワード）クリックでも同じ絞り込みに連動し、ページ上部へスムーススクロール。
- フィルタ結果が0件のセクションはセクションごと非表示。
- 選択中のピル／チップはアクティブ表示で状態を可視化。

#### 6.1.4 コンテンツセクション
本文（メインカラム）に以下のセクションを縦に配置。各セクションは見出し（英語リード＋日本語）＋「VIEW ALL」（一覧へ）＋カードグリッド。

| セクション | 内容 | カード要素 | VIEW ALL 遷移先 |
|---|---|---|---|
| INTERVIEW（編集部インタビュー） | 職員/管理職/座談会/技術職/新卒 | サムネ＋コンテンツタイプ＋タイトル＋自治体（アバター＋名称） | `list.html?cat=interview` |
| MOVIE（動画コンテンツ） | 番組型動画 | サムネ＋再生アイコン＋再生時間＋タイトル | `list.html?cat=movie` |
| COLUMN（編集部コラム） | 面接・キャリア・自己PR等 | サムネ＋カテゴリー＋タイトル＋公開日 | `list.html?cat=column` |

- カードクリックで各詳細ページ（インタビュー＝`iv-*`、コラム＝`col-*`、動画＝`movie.html`、汎用＝`article.html`）へ遷移。
- 各カードは `data-tags`（キーワード）／`data-cat`（記事/動画）を保持し、絞り込み対象となる。

#### 6.1.5 サイドバー
- **ランキング**（PICK UP）：人気記事 上位5件（順位番号＋タイトル）。
- **広告枠**：縦長バナー（300×600）×1、レクタングル（300×250）×3。広告は外部配信を想定（枠のみ定義）。

#### 6.1.6 フッター
- ブランド表記、ナビ（求人サイト／会員登録／特集／インタビュー／動画／コラム／編集方針／お問い合わせ）、著作権表記。

### 6.2 一覧ページ（list.html）
- クエリ `?cat=interview | movie | column` でカテゴリー別一覧を表示。
- トップと同様のカード形式。ページネーションまたは無限スクロール、絞り込み（カテゴリー／キーワード）を備える。

### 6.3 記事詳細（article / iv-* / col-* / ガイド記事）
- 記事本文（見出し・本文・画像）、コンテンツタイプ・タグ、公開日、（インタビューは）対象自治体・職員情報。
- 関連記事・関連求人（該当自治体の求人）への導線。
- SNSシェア、会員登録／求人サイトCTA。
- パンくず（メディアトップ＞カテゴリー＞記事）。

### 6.4 動画ページ（movie.html）
- 動画プレイヤー（再生時間表示）、タイトル・説明・タグ、関連動画・関連求人導線。

### 6.5 ガイドコンテンツ群（SEO）
- 公務員試験・転職テーマの網羅記事（試験内容/流れ/区分/過去問/年代別/給与/メリット 等）。検索流入の獲得が主目的。
- 内部リンクで関連記事・求人・会員登録へ送客。

## 7. 非機能要件

| 区分 | 要件 |
|---|---|
| レスポンシブ | PC（〜1440px）・タブレット・スマホに対応。メイン＋サイドの2カラムはスマホで1カラムに段組み変更。 |
| SEO | 適切なタイトル・メタディスクリプション、構造化データ（Article/VideoObject/BreadcrumbList）、正規URL、サイトマップ、内部リンク設計。 |
| パフォーマンス | 画像の遅延読み込み・最適化（WebP等）、コアウェブバイタル配慮、スライド/フィルタの軽量実装。 |
| アクセシビリティ | WCAG AA 相当（代替テキスト、キーボード操作、コントラスト、`aria-label`、フォーカス可視）。 |
| ブラウザ | 主要モダンブラウザ最新版＋1世代。 |
| 計測 | アクセス解析（PV/UU/流入元/回遊/CV）・イベント計測（動画再生、CTAクリック、PR枠クリック）。 |
| 広告 | 広告枠の安全な表示（レイアウトシフト抑制）、PR表記の明確化（景品表示・ステマ規制配慮：要法務確認）。 |

## 9. データ要件（主なエンティティ・概念）

> 本モックは静的HTML（ダミーデータ）。本実装ではCMS/DBで以下を保持する想定。

- **コンテンツ（記事/動画）**：ID、タイトル、種別（記事/動画）、コンテンツタイプ、本文/動画URL、サムネイル、公開日、ステータス、PR有無、関連自治体、関連求人、キーワードタグ（複数）、（動画）再生時間。
- **タグ（キーワード）**：ID、名称、表示順。
- **特集バナー**：ID、画像、リンク先、PR有無、掲載期間、表示順。
- **ランキング**：集計ロジック（PV等）に基づく上位N件（手動ピックアップ枠の併用可）。
- **広告枠**：枠ID、サイズ（300×600 / 300×250）、配信設定。
- **自治体**：名称、ロゴ/アバター、求人サイト側マスタと連携。

## 10. 運用・編集要件（概要）

- 編集部による記事・動画・コラム・インタビューの企画／入稿／公開。
- 特集（PR含む）の枠管理・掲載期間管理。PR枠は契約に基づき `PR` 表記を必須とする。
- カテゴリー・キーワードタグの付与ルール（最低/推奨タグ数、表記統一）。
- ランキングの集計・更新頻度、手動ピックアップの運用ルール。
- 公開承認フロー（下書き→レビュー→公開）。※対象外章だが連携前提として明記。

---

## 11. 制約・前提

- 本書はモック（`user/media/`）の挙動を基に作成。最終仕様は事業要件・運用体制により変動する。
- 掲載する自治体名・職員情報・コンテンツはモック上ダミー。実運用では実データ／許諾に基づく（個人情報・肖像・著作権の取扱いは要法務確認）。
- PR表記・広告表示は景品表示法・ステルスマーケティング規制等の遵守が前提（専門家確認）。
- 会員/求人/広告/動画の各基盤は外部依存。連携IF・SLAは別途定義。
